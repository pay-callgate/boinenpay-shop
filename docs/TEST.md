- **목적:** 모듈/Phase 완료 시 기능 정합성, Multi-Tenancy 격리, 역할별 접근, 사용자 시나리오 E2E, 엣지 케이스 검증.
- **수준:** 단위(Jest/Vitest), 통합(API·DB), E2E(Playwright/Cypress), 수동(UI/반응형).
- **환경:** Local (localhost:3000), Staging, Production.

## 2. 테스트 착수 가능 범위

### 완료 Phase (테스트 가능)

- Phase 0~6, 8: 인프라, 파트너 온보딩, 상품·거래처·소속 매칭, 쇼핑몰, 주문, 마이페이지, 최근 본 상품.
- Phase 10 (CallCloud 070): 부분 완료(브라우저·자동화 플로우 구현 후 단계별 검증).

### 파트너 사용자 테스트 항목

- SNS 로그인, 기업 등록, 상품/카테고리/거래처 CRUD, 링크 생성·복사, 070 설정, 주문 목록·상세·상태·송장·엑셀, 재고 관리, 알림.

### 거래처 사용자 테스트 항목

- 링크 진입·자동 매칭, 소속 검색, 쇼핑몰 메인·PLP·PDP, 장바구니·주문서·결제, 마이페이지·배송지·관심상품.

*(상세 범위·미구현 Phase는 원본 TEST_READINESS_REVIEW.md 참고)*

## 3. 실행 체크리스트

### 사전 준비

- 환경: `npm run dev`, `.env.local` (Supabase, NextAuth, SMTP/Solapi).
- 테스트 데이터: 파트너·거래처·카테고리·상품·주문 등 시드 또는 OAuth 동적 생성.

### 시나리오 A: 파트너 사용자

- 로그인·온보딩, 상품 관리(등록·이미지·재고), 거래처 관리(등록·링크·070), 주문 관리(필터·상세·상태·송장·엑셀), 재고 관리, 로그아웃. (약 29개 체크)

### 시나리오 B: 거래처 사용자 (링크 진입)

- 링크 진입·자동 매칭, 쇼핑몰 탐색(배너·PLP·PDP), 장바구니·주문·결제, 마이페이지. (체크 항목 다수)

### 시나리오 C: 거래처 사용자 (일반 진입)

- 소속 기업 찾기 팝업·검색·매칭 후 쇼핑몰 이용.

*(전체 체크 항목·결과 기록란은 원본 TEST_CHECKLIST.md 참고)*
