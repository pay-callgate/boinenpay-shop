# 결제 완료 페이지 "결제 정보가 없습니다" 오류 원인 및 수정

**일자:** 2026-03-11  
**상황:** 결제는 정상 완료되었으나 주문 완료 페이지에서 "결제 정보가 없습니다. 주문 내역에서 결제 상태를 확인해 주세요." 표시.

---

## 1. 원인 분석

### 1.1 주요 원인: returnUrl 리다이렉트 시 `cgTid` 미전달

- 우리 **returnUrl**: `https://도메인/wooribugo/knauto/order/complete?orderId=xxx`
- 결제 완료 후 ViewPay가 이 URL로 **리다이렉트**할 때, **결제 거래 ID(cgTid/tid)** 를 쿼리에 붙이지 않거나, 다른 이름/위치로 보낼 수 있음.
- 주문 완료 페이지는 **orderId + cgTid** 가 있을 때만 `GET /api/payment/viewpay/complete` 를 호출함.
- **cgTid가 없으면** complete API를 호출하지 않고, 기존에는 "결제 정보가 없습니다" **에러 메시지**를 표시함 → 결제는 된 상태인데 오류처럼 보임.

### 1.2 Vercel 로그의 401 (참고)

- `/api/cart` 에서 `401 - 세션 없음` 이 나오는 것은, **ViewPay 결제창(외부 도메인)에서 우리 사이트로 돌아올 때** 쿠키/세션 이슈가 있을 수 있음을 의미함.
- 이번 "결제 정보가 없습니다" 메시지의 **직접 원인**은 **cgTid 부재**이며, 401은 별도로 세션/쿠키 설정 검토 시 참고하면 됨.

---

## 2. 수정 사항 (반영 완료)

### 2.1 결제 거래 ID 다중 수용

- **쿼리 파라미터**: `cgTid`, `tid`, `tId`, `paymentId`, `transactionId`, `cg_tid` 중 **먼저 있는 값**을 사용.
- **URL hash**: `#cgTid=xxx` 또는 `#tid=xxx` 형태로 넘어오는 경우에도 파싱해 사용.
- ViewPay가 쿼리 대신 hash로 넘기거나, 파라미터 이름이 다를 수 있음을 전제로 처리.

### 2.2 cgTid 없을 때 UX 변경

- **기존**: "결제 정보가 없습니다. 주문 내역에서 결제 상태를 확인해 주세요." (에러 톤)
- **변경**: "결제가 완료되었을 수 있습니다. 주문 내역에서 결제 상태를 확인해 주세요." (안내 톤)
- **[주문 내역]** 버튼을 **첫 번째**로 배치해, 결제 여부 확인을 우선 유도.

---

## 3. ViewPay 측 확인 권장

- returnUrl 리다이렉트 시 **어떤 파라미터를 어떤 이름으로** 붙이는지(예: `cgTid`, `tid` 등) 연동 규격서 또는 기술 지원으로 확인.
- 가능하면 **성공 시** `returnUrl?orderId=xxx&cgTid=yyy` 형태로 쿼리에 포함되도록 요청하면, 완료 페이지에서 자동으로 complete API 호출 후 "결제가 완료되었습니다" 화면을 보여줄 수 있음.

---

## 4. 세션 401 관련 (추가 점검)

- 리다이렉트 후 `/api/cart` 401이 반복되면:
  - **NEXTAUTH_URL**: Vercel 배포 URL과 일치하는지.
  - **쿠키 도메인 / SameSite**: 외부(ViewPay) → 우리 도메인 이동 시 쿠키 전송 여부.
  - NextAuth `useSecureCookies`, `cookieOptions` 등 프로덕션 설정 검토.
