# 데모 시연 및 전수 검사 체크리스트

**작성일:** 2025-02-10  
**목적:** 내부 직원 데모 시연 전 최종 점검 및 거래처 쇼핑몰 ↔ 마스터 템플릿 동기화·신규 거래처 등록 전수 검사

---

## 1. 거래처 쇼핑몰 ↔ 마스터 템플릿 100% 동기화 요약

| 항목 | 마스터 템플릿 (미리보기) | 거래처 쇼핑몰 (기아자동차, LG전자 등) | 동기화 |
|------|--------------------------|----------------------------------------|--------|
| **진입 URL** | `/{subdomain}` (예: `/testpartner`) | `/{subdomain}/{clientSlug}` (예: `/testpartner/knauto`) | ✅ |
| **레이아웃** | `ShopLayout` → SmartHeader + SideMenu + ShopBottomNav | `ShopGlobalLayout` → 동일 SmartHeader + SideMenu + ShopBottomNav | ✅ |
| **헤더** | SmartHeader (MASTER TEMPLATE 표시) | SmartHeader (거래처 로고/이름) | ✅ 동일 컴포넌트 |
| **하단 네비** | ShopBottomNav (홈, 카테고리, 최근본, 마이페이지) | 동일 | ✅ |
| **메인 콘텐츠** | ShopMainHome (카테고리 탭, 상품 그리드) | ShopMainHome 동일 | ✅ |
| **주문/장바구니** | 차단 (alert) | 허용 | ✅ 의도적 차이 |
| **데이터** | partner from API, client=null | partner/client from 서버 레이아웃 | ✅ |
| **디자인** | PRIMARY #D6A8E0, max-w-[430px], HEADER 56px, BOTTOM 64px | 동일 | ✅ |

- **마스터 템플릿**: `app/[subdomain]/page.tsx` → `ShopLayout` + `ShopMainHome` (partner만, client=null).  
- **거래처 쇼핑몰**: `app/[subdomain]/[clientSlug]/layout.tsx` → 서버에서 partner/client 조회 후 `ShopGlobalLayout`에 전달, 모든 자식 페이지는 `useShopTemplate()`으로 동일 Context 사용.

## 2. 신규 거래처 등록 전수 검사 시나리오

### 2.1 어드민 — 거래처 등록

- [ ] **로그인**  
  - 어드민 로그인 후 `/admin` 대시보드 접근 가능한지 확인.
- [ ] **거래처 등록 모달**  
  - [거래처 등록] 클릭 → 모달에서 **거래처명**, **Slug**(영문, 고유), **담당자명**, **담당자 연락처**, **담당자 이메일**, **대표자명**, **대표 이메일**, **대표 연락처**, **주소** 등 입력 후 저장.
- [ ] **중복 Slug**  
  - 이미 존재하는 slug로 저장 시 "동일한 slug가 이미 존재합니다." (409) 메시지 확인.
- [ ] **저장 후 목록 반영**  
  - 저장 성공 시 모달 닫힘, 거래처 목록에 새 거래처가 바로 표시되는지 확인.
- [ ] **거래처 수정**  
  - 기존 거래처 수정 시 **대표자명/대표 이메일/대표 연락처**가 폼에 채워지는지, 수정 후 저장 시 반영되는지 확인.

### 2.2 어드민 — 거래처/링크 관리 (070 연동)

- [ ] **링크 주소**  
  - 거래처별 주문 전용 URL 표시 (예: `http://localhost:3000/testpartner/{slug}`). [복사하기] 동작 확인.
- [ ] **070 연동하기**  
  - [070 연동하기] 클릭 시 070 연동 모달 오픈.  
  - **관리자명**, **관리자 이메일**, **관리자 전화번호**가 해당 거래처의 **담당자명**, **담당자 이메일**, **담당자 연락처**로 자동 세팅되는지 확인.
- [ ] **고객사 Link 안내 메시지**  
  - Link 안내 모달에서 [메시지 내용 수정] 버튼이 우측 "필요 시 내용을 수정할 수 있습니다." 문구 오른쪽에 있는지 확인.

### 2.3 쇼핑몰 — 마스터 템플릿 미리보기

- [ ] **URL**  
  - `/{subdomain}` (예: `http://localhost:3000/testpartner`) 접속.
- [ ] **헤더**  
  - "MASTER TEMPLATE" 표시, 햄버거/검색/장바구니 아이콘 동작.
- [ ] **카테고리 탭**  
  - 가로 슬라이딩(스와이프) 가능한지 확인. **건강기능식품** 등 신규 카테고리 노출 여부 확인.
- [ ] **상품 그리드**  
  - 카테고리별 섹션·상품 노출. [MORE VIEW] 동작.
- [ ] **장바구니/주문**  
  - 장바구니 담기 또는 주문 시 "마스터 템플릿 미리보기 상태에서는..." alert로 차단되는지 확인.

### 2.4 쇼핑몰 — 거래처 전용 URL (신규 등록 거래처)

- [ ] **URL**  
  - `/{subdomain}/{새거래처 slug}` (예: `http://localhost:3000/testpartner/newclient`) 접속.
- [ ] **헤더**  
  - 거래처 로고 또는 거래처명 표시. (로고 없으면 이름만 표시)
- [ ] **메인**  
  - 마스터 템플릿과 동일한 ShopMainHome (카테고리 탭, 상품 그리드). 카테고리 슬라이딩 동작.
- [ ] **유효하지 않은 slug**  
  - 존재하지 않는 slug (예: `/testpartner/invalidslug`) 접속 시 "해당 거래처 링크를 찾을 수 없습니다." 안내 및 [파트너 홈으로 이동] 버튼 동작 확인.
- [ ] **마이페이지(비로그인)**  
  - 주문/결제 등 제한 시 OrderGuard 등으로 로그인 유도 메시지가 나오는지 확인.
- [ ] **로그인 후**  
  - 거래처 전용 URL에서 로그인 시 자동 매칭(소속 기업 등록) 후 주문/장바구니 이용 가능한지 확인.

### 2.5 쇼핑몰 — 공통 플로우 (기아자동차, LG전자 등 기존 거래처)

- [ ] **홈**  
  - 배너, 카테고리 탭, 카테고리별 상품. 카테고리 가로 슬라이딩.
- [ ] **상품 목록**  
  - `/products`, `/products?category=xxx`. 카테고리 필터·정렬·검색.
- [ ] **상품 상세**  
  - 이미지, 가격, 옵션, 장바구니/구매. 관심상품 담기.
- [ ] **장바구니**  
  - 담기 후 장바구니 페이지에서 수량 변경, [주문하기] → 체크아웃 이동.
- [ ] **체크아웃**  
  - 배송지, 결제 수단 등 입력 후 주문 완료까지 (데모 시 결제는 mock 가능).
- [ ] **마이페이지**  
  - 프로필, 주문 조회, 관심상품, 배송지. (거래처 미매칭 시 "거래처를 찾을 수 없습니다" 등 안내)
- [ ] **최근 본 상품**  
  - 상품 상세 조회 후 최근 본 상품 목록 노출.
- [ ] **사이드 메뉴**  
  - 햄버거 메뉴 → 로고/로그아웃, 퀵 메뉴, 카테고리 아코디언. 카테고리 클릭 시 해당 목록으로 이동.

### 2.6 카테고리

- [ ] **신규 카테고리 노출**  
  - 어드민에서 추가한 카테고리(예: 건강기능식품)가 쇼핑몰 메인·상품 목록·사이드 메뉴에 F5 새로고침 후 노출되는지 확인.
- [ ] **카테고리 탭 슬라이딩**  
  - 메인/상품 목록 페이지에서 카테고리 바 가로 스와이프 시 슬라이딩 동작.

## 3. 소스 코드 최종 점검 반영 사항

- **유효하지 않은 거래처 slug 처리**  
  - `app/[subdomain]/[clientSlug]/page.tsx`: 파트너는 있으나 해당 `clientSlug`에 해당하는 거래처가 없을 때(단, `_preview` 제외) "해당 거래처 링크를 찾을 수 없습니다." 안내 및 [파트너 홈으로 이동] 버튼 표시하도록 추가됨.  
  - 마스터 템플릿 미리보기는 반드시 `/{subdomain}/_preview` 또는 `/{subdomain}` 사용.

- **동기화 정리**  
  - 마스터 템플릿: `ShopLayout` + `ShopHeader`(내부적으로 SmartHeader 사용) + 동일 SideMenu/ShopBottomNav.  
  - 거래처 쇼핑몰: `ShopGlobalLayout` + SmartHeader + SideMenu + ShopBottomNav.  
  - 동일 상수(HEADER_HEIGHT, BOTTOM_NAV_HEIGHT, PRIMARY #D6A8E0) 및 동일 ShopMainHome/공통 shop 컴포넌트 사용으로 100% 동기화 유지.

## 4. 데모 시 연출 팁

1. **진입 순서**  
   - 어드민 → 거래처 등록 → 링크/070 확인 → 쇼핑몰 URL로 접속(마스터 미리보기 → 거래처 전용) 순서로 보여주면 흐름이 명확합니다.
2. **마스터 vs 거래처**  
   - `/{subdomain}` 과 `/{subdomain}/{clientSlug}` 를 나란히 비교하며 헤더(MASTER vs 거래처 로고/이름)만 바뀌고 나머지 UI가 동일함을 강조할 수 있습니다.
3. **신규 거래처**  
   - 데모 직전에 한 건 등록해 두고, 등록 직후 쇼핑몰 URL로 접속해 바로 노출되는 것을 보여주면 효과적입니다.

이 체크리스트를 기준으로 전수 검사 후 데모 진행하시면 됩니다.
