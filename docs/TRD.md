## 2. 아키텍처 개요

### 2.1 Multi-Tenancy URL 전략

```
https://{Partner_Subdomain}.{Platform_Domain}/{Client_Slug}
```

| 계층 | 추출 방식 | DB 조회 |
|------|-----------|---------|
| Partner_Subdomain | Host 헤더 또는 URL 파싱 | partners 테이블 |
| Client_Slug | Path 첫 번째 세그먼트 | clients 테이블 |
| client_source_id | Slug → clients.id | 세션/쿠키 저장, 주문 시 사용 |

### 2.2 세션/쿠키 처리

- 접속 시 `client_source_id` (또는 `client_slug`) 저장
- 주문 생성 시 자동으로 `client_id` FK 연결
- 파트너 식별은 Subdomain으로 처리

### 2.3 도메인 루트 라우팅 규칙

| 접속 URL | 처리 |
|----------|------|
| `https://shopping.com/`, `https://www.shopping.com/` | `https://www.callgate.com/index.html` 리다이렉트 |
| `https://{subdomain}.shopping.com/admin` | 파트너 어드민 (로그인·대시보드 등). **라우팅:** app/[subdomain]/admin/. 미인증 시 리다이렉트. **접근:** user.role == PARTNER_ADMIN AND user.company_id == subdomain의 파트너 |
| `https://{subdomain}.shopping.com/`, `https://www.{subdomain}.shopping.com/` | 파트너 메인 마스터 템플릿 (주문 불가, 브라우징만) |
| `https://{subdomain}.shopping.com/login` | 쇼핑몰 로그인 (거래처 사용자. 주문 시도 시 로그인 유도) |
| `https://{subdomain}.shopping.com/{clientSlug}` | 거래처 주문 전용 메인 (CI 이미지/텍스트 반영) |

**도메인에 따른 역할 자동 분기:** 진입 URL로 가입 목적을 추정합니다. `{subdomain}.shopping.com/admin` → 파트너(판매자) 가입 의도 → 기업 등록 페이지. `{subdomain}.shopping.com/login` → 거래처 직원(구매자) 가입 의도 → 소속 매칭 로직. (PRD §4.2-1 참조)

**거래처 어드민:** 별도 /admin 경로 없음. `/{subdomain}/{clientSlug}/mypage` 내에서 CLIENT_ADMIN일 때 직원 승인·전용 주문 내역 등 조건부 노출.

**Middleware 보안 (파트너 어드민):** /admin 진입 시 (1) 로그인 여부 (2) user.role == PARTNER_ADMIN (3) user.company_id와 현재 subdomain의 파트너 ID 일치 검증. 불일치 시 403 또는 메인 리다이렉트.

### 2.4 개발 환경 (localhost)

**로컬 테스트:** `http://localhost:3000/{subdomain}/...` 로 접속해서 테스트하세요. (예: `http://localhost:3000/yenmidang/samsungelec`). Next.js Middleware가 경로 첫 세그먼트를 subdomain으로 인식합니다.

| 접속 | 처리 |
|------|------|
| `localhost:3000/` | `localhost:3000/yenmidang/` 리다이렉트 (기본 파트너) |
| `localhost:3000/{subdomain}/admin` | 파트너 어드민 (예: yenmidang/admin). 프로덕션과 동일 규칙 |
| `localhost:3000/{subdomain}/` | 파트너 메인. subdomain을 경로 첫 세그먼트로 시뮬레이션 |
| `localhost:3000/{subdomain}/{clientSlug}` | 거래처 메인 |
| `localhost:3000/{subdomain}/{clientSlug}/mypage` | 마이페이지 (거래처 어드민 확장 포함) |
| `localhost:3000/{subdomain}/login` | 쇼핑몰 로그인 (거래처 사용자. 예: yenmidang/login) |

## 5. 데이터 모델 (핵심 엔티티)

### 5.1 파트너/거래처

- **partners**: 파트너사 (subdomain, 사업자정보 등)
- **clients**: 거래처 (slug, partner_id FK, 담당자, call_070_connected). 전용 URL은 `https://{partners.subdomain}.{domain}/{clients.slug}` 로 파생

### 5.2 상품

- **products**: 상품 마스터 (partner_id FK)
- **product_categories**: 카테고리 (New, Best Seller 등)
- **product_options**: 옵션 (사이즈, 색상 등)
- **product_inventory**: 재고, Safety Stock

### 5.3 주문

- **orders**: 주문 (client_id FK, order_no, status 등)
- **order_items**: 주문 상품 (product_id, quantity, price 등)
- **order_status_history**: 상태 이력

### 5.4 사용자

- **users**: 회원 (auth)
- **user_clients**: 거래처 직원 ↔ 거래처 1:1 매핑 (user_id UK)
- **partner_admins**: 파트너사 관리자 매핑

## 7. 070 콜 시스템 연동

### 7.1 연동 플로우

1. 거래처별 주문 전용 URL 생성 (`https://{subdomain}.shopping.com/{clientSlug}`)
2. 거래처 관리 화면 링크 주소 옆 **[070번호 연결하기]** 버튼 클릭
3. **입력 팝업** 오픈 → `client_call_070_configs` 테이블에 저장할 필드 입력
4. [등록] 클릭 시 **Chrome Selenium** 기반 CallCloud 백오피스 자동화 수행
5. `https://backoffice.callcloud.kr:27091/login` 접속 → 로그인 → [신규 고객사 등록] 클릭 → 폼 자동 입력
6. 연동 완료 시 `callcloud_registered=true`, `clients.call_070_connected=true` 반영 → "070번호 연동이 완료되었습니다." Alert

### 7.2 입력 팝업 (070번호 연결하기) — 화면 스펙

**레이아웃:** 모달/다이얼로그. 제목 "070번호 연동"

|------|------|------|-------------|------|
| 고객사명 | 고객사명 | ✓ | clients.name 연동 (예: 연미당) | |
