# Rules 폴더 - 프로젝트 규칙 관리

본 폴더는 **콜링크 쇼핑몰 플랫폼** 프로젝트 진행 시 지켜야 할 모든 규칙을 체계적으로 관리하기 위한 공간입니다.

## 폴더 목적

- 프로젝트 전반에 적용되는 **규칙(Rules)** 을 문서화
- 개발, 리뷰, 협업 시 일관된 기준 제공
- **지속적 업데이트**: 프로젝트 진행에 따라 규칙 추가·수정

## 규칙 파일 구성

| 파일 | 용도 |
|------|------|
| **PROJECT_RULES.md** | 프로젝트 전반 규칙, 우선순위, 금지 사항 |
| **CODING_STANDARDS.md** | 코딩 스타일, 네이밍, 파일 구조 |
| **ARCHITECTURE_RULES.md** | 아키텍처 결정, 패턴, 제약 사항 |

## 사용 방법

1. **규칙 추가/수정**: 해당 규칙 파일을 직접 편집
2. **새 규칙 파일**: 새로운 주제의 규칙이 필요하면 새 md 파일 생성
3. **Cursor 연동**: `.cursor/rules/` 에서 본 폴더 규칙을 참조하도록 설정 가능
4. **커밋**: 규칙 변경 시 의도와 변경 사유를 커밋 메시지에 명시

## 업데이트 주기

- **필요 시 즉시**: 새로운 결정·제약이 생길 때마다
- **스프린트/마일스톤별**: 정기 점검 및 정리

---

*규칙은 팀의 합의 하에 유연하게 수정됩니다. 단, 변경 시 팀 공유를 원칙으로 합니다.*
